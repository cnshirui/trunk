Deploying Rack-based Apps

Heroku supports Rack and Rack-based web frameworks like Sinatra, Merb, Ramaze, and Camping.

To run a Rack-based app, include a rackup file named config.ru in the app’s root directory. The config.ru file convention has become common, so most existing Rack applications should not require changes to deploy to Heroku.

  • Creating and Deploying Pure Rack Apps
  • Sinatra
  • Merb
  • Ramaze

Creating and Deploying Pure Rack Apps

First, create a new directory and write a simple config.ru file:

$ mkdir hello
$ cd hello
$ cat > config.ru
run lambda { |env| [200, {'Content-Type'=>'text/plain'}, StringIO.new("Hello World!\n")] }
[Ctrl-D]

Test it locally:

$ gem install rack
$ rackup -p 9292 config.ru &
$ curl http://localhost:9292
Hello World!
$ kill %1

Deploy to Heroku:

$ git init
$ git add .
$ git commit -m 'pure rack app'
$ heroku create
$ git push heroku master

The app is now deployed to Heroku. Test by executing heroku open or by visiting your app’s URL in your browser. You should see Hello, World!.

Sinatra

hello.rb:

1.require 'sinatra'
2. 
3.get '/' do
4.  "Hello World!"
5.end

config.ru:

1.require 'hello'
2.run Sinatra::Application

Merb

The following config.ru file can be used for all Merb apps.

1.require 'merb-core'
2.Merb::Config.setup(:merb_root => ".", :environment => ENV['RACK_ENV'])
3.Merb.environment = Merb::Config[:environment]
4.Merb.root = Merb::Config[:merb_root]
5.Merb::BootLoader.run
6. 
7.use Merb::Rack::Static, Merb.dir_for(:public)
8.run Merb::Rack::Application.new

Ramaze

hello.rb:

1.require 'ramaze'
2. 
3.class MainController < Ramaze::Controller
4.  def index
5.     "Hello World!"
6.  end
7.end

config.ru:

1.require 'hello'
2.Ramaze.trait[:essentials].delete Ramaze::Adapter
3.Ramaze.start!
4.run Ramaze::Adapter::Base

Camping

hello.rb:

01.require 'camping'
02. 
03.Camping.goes :Hello
04. 
05.module Hello::Controllers
06.  class Index < R '/'
07.     def get
08.        render :hello
09.     end
10.  end
11.end
12. 
13.module Hello::Views
14.  def hello
15.     p  "Hello World!"
16.  end
17.end

config.ru:

1.require 'hello'
2.run Rack::Adapter::Camping.new(Hello)
Camping 2.0 does not require the Rack adapter; use `run Hello` instead.

Using ActiveRecord

Rails or Merb will automatically read config/database.yml, which is written with the PostgreSQL config information at compile time. For other frameworks, put this code into your application to access the database with ActiveRecord:

1.require 'activerecord'
2. 
3.dbconfig = YAML.load(File.read('config/database.yml'))
4.ActiveRecord::Base.establish_connection dbconfig['production']

For local development, you should create a config/database.yml file in your which contains a pointer to your local database. A good technique is to copy config/database.yml from a blank Rails app, which uses the SQLite database db/development.sqlite3.

Using DataMapper or Sequel

DataMapper and Sequel both use database URLs instead of config hashes. You could read config/database.yml and construct the URL, but an easier way is to use the DATABASE_URL environment variable set by Heroku.

For DataMapper:

1.require 'rubygems'
2.DataMapper.setup(:default, ENV['DATABASE_URL'] || 'sqlite3://my.db')

For Sequel:

1.require 'sequel'
2.Sequel.connect(ENV['DATABASE_URL'] || 'sqlite://my.db')

This defaults to a SQLite database named my.db when no DATABASE_URL is set. You can also make use of the DATABASE_URL environment variable when running locally or on another host. For example:

$ DATABASE_URL='mysql://user:pass@localhost/mydb' thin start --rackup config.ru
